User First Design

  1. As a new user, I should be able to register for an account.

    • User visits homepage.
    • User clicks "Sign Up" button.
    • User fills out a "Create An Account" form.
    • User clicks "Create Account" button.
    • User is taken to their user page.

  2. As a returning user, I should be able to scroll through my feed.

    • User visits homepage.
    • User enters username and password.
    • Users clicks "Log In".
    • User is taken to their feed page.
    • User can scroll to view posts.

  3. As a returning user, I should be able to make a post.

    • User visits homepage.
    • User enters username and password.
    • Users clicks "Log In".
    • User is taken to their feed page.
    • User clicks "New Post" button.
    • User enters post text and clicks "Post" button.
    • User is taken back to their feed page.

  4. As a returning user, I should be able to click on a hashtag and view posts for it.

    • User visits homepage.
    • User enters username and password.
    • Users clicks "Log In".
    • User is taken to their feed page.
    • User clicks on a hashtag in a post on the feed.
    • User is taken to a feed page of all posts from all users with the selected hashtag.

  5. As a returning user, I should be able to view another users profile.

    • User visits homepage.
    • User enters username and password.
    • Users clicks "Log In".
    • User is taken to their feed page.
    • User clicks on a username.
    • User is taken to selected username's page.

Data First Design

  1. Users

    Data needed

    Each user will need a unique username and a password. It is important that the password is stored securely. Each user will need to store a list of posts. Each user will store a list of followERs and followEEs. Additionally, users may want to store some contact information and a profile picture.

    RESTful routes pages

    1. Create - Show page with message.
    2. Show - Username, profile picture, and users posts shown automatically, with options to see follower/followee list. If logged in as user, can see user feed.
    3. Update - Show page with message.
    4. Delete - Index page with message.
    5. Edit - Same form as New page.
    6. New - Form containing fields for username, password, profile picture, and other information (email?).
    7. Index - A list of all usernames (with profile picture possibly).

  2. Posts/Messages

    Data needed

    Posts need to keep track of their textual content and the user that created them, as well as metadata including (but not limited to) date/time posted.

    RESTful routes pages

    1. Create - Show page with message.
    2. Show - Text of post accompanied by username and profile picture of poster, with mentions and hashtags highlighted or denoted some other way.
    3. Update - Show page with message.
    4. Delete - Index page with message.
    5. Edit - Same as New form.
    6. New - Form with text field and submit button.
    7. Index - A list of all posts (filterable by user, mentioned user, hashtag, etc. possibly)
